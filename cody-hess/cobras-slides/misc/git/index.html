git dvcs

distributed version control system

source code management

`git` is how you collaborate

`git` is how you go back in time

you *must* become comfortable with `git`

push your work every day

git status

tells you which changed files are tracked / untracked

tracked means these changes will be a part of your next commit

tracked files are referred to as "staged"

`git status` just gives you information; it doesn't *do* anything

git checkout -b

`git checkout <branch_name>` switches branches

`git checkout -b <branch_name>` creates a branches and switches to it

the very first thing you do every day is `git checkout -b <branch_name>` to create your own branch on that day's repo

a "repo" is a repository of code

a "branch" is a set of changes independent of other branches

git add

`git add <name_of_file>` stages a file for the next commit

`git add .` is a shortcut to stage all changes at, and below, the present working directory

git commit

`git commit` logs all of your staged changes

`git commit` will open a text editor for you to enter your commit message

your commit message should be a short description of your changes

`git commit -m '<your_commit_message>'` is a shortcut to commit with a message in a single command

git push

`git push <location> <branch>` copies your branch to another location

for now, your location will always be "origin"

for now, "origin" will always mean GitHub

do this before you leave every day

do this before you leave every day !

do this before you leave every day !!!

summary

`git checkout -b <your_name>` for every new repo

`git add <file_name>`, `git commit -m '<your_commit_message>'`, frequently while you are working

`git push origin <your_branch_name>` before you leave
